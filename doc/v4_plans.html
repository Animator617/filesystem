Boost Filesystem V4 Plans

The objectives of Boost Filesystem Version 4:

  • Bring Boost.Filesystem into much closer conformance with the C++17 standard library's section 27.10, File systems [filesystems].
  • Preserve as much existing Boost.Filesystem user code as possible without compromising the other objectives.
  • Document breaking changes.
  • Where existing user code will be broken, provide tools to aid user migration.
  • Modernize the library's codebase.
  • Close lots of bug reports in the process.

Whether or not there will be a configuration macro to continue support for version 3 for several more releases is still undecided. Doing that during the V2 to V3 transition was not a pleasant experience and greatly slowed development. OTOH, there should be much less breakage going from V3 to V4.

 

Modernizing the library's codebase

There are two aspects of this:

  1. Refactor some of the implementation code. Should not impact those who use Boost.Build to build the library.
  2. Use widely available C++11 features. This impacts users of older compilers.

C++11 features that may be used

  • auto, decltype, class enums, lambdas, rvalue references.
